4. Critical path

Computing task slack and showing critical path in chart

04-
Critical_path
.xml
  • See documentation for Critical path.
  • The critical path can be shown if a column is assigned to

    GanttSlack

    .
  • The Slack value means how much can be the task moved to future (delayed) to not affect project finish date.
    The tasks that cannot be moved at all are critical and lay on critical path.
    The critical tasks are colored in different color (orange by default) and it is also possible to filter by them to show only the critical path.
  • Attribute

    GanttMinSlack

    sets minimal value of the Slack to be the task included in critical path.
  • Task with Slack value less than

    GanttErrSlack

    are marked as error and colored in different color (red by default).
    The GanttErrSlack value should be less or equal to GanttMinSlack.
  • The finish date is end date of the most right task or it can be defined by

    GanttFinish

    .
  • If set

    GanttSeparateSlack

    ='1' the Slack value is calculated for every group of tasks separately.
    The group of tasks are all tasks linked with some dependency.
    In this case is the global GanttFinish ignored.
